Welcome to Viewport v1.0

A simplified way to test responsive website layouts at common resolutions.

Recently I've been doing some responsive design work and I found that checking screen resolutions was a bit of a pain. Dragging the window size works adequately for watching responsive styles "step-down", but it's tricky so get it pixel perfect and my browser windows won't go smaller than 400px wide.

I couldn't find any free simple tools to do exactly what I wanted (including hiding the scrollbar), so I decided to build my own. This is what I made - I have found it to be quite useful and I hope you will too.

Browser specific features: WebKit browsers (Chrome and Safari) allow us to hide the scrollbar, but stll allow scrolling. I recommend using Chrome because Safari's iframes tend to misbehave for me. I don't recommend using this on a mobile device, but feel free to try it.

Frequently Asked Questions

Q: Why doesn't it work on my site?

A: Viewport is really just a resizable iframe with a pretty border. Any sites that use frame-busting techniques will not work with this application.

Q: Why does the Share Link feature use the original URL entered?

A: In an effort to thwart cross site scripting attack (XSS), browsers have set security limitations on the data that is available when the parent page and the iframe page are from different domains. The URL enetered is the best we can do.

Q: Can you make scrollbars hideable in non-WebKit browsers?

A: Yes, we can do that, but we don't have the ability to detect when we should. Due to iframe security limitations, we cannot detect if the scrollbar is being displayed. If we try to hide the scrollbars when they are not being displayed, it will hide some of the page.

Q: I don't like your implementation, what else can I use?

A: What a shame that you don't like it. You could try todd's jResize or maybe The Responsinator.
